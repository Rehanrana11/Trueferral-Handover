🚨 THREE KILL CRITERIA (HARD STOPS)

KILL #1: CORE LOOP BROKEN (RECEIPT → SHARE → CLOSURE)

Hypothesis being tested:
Users will repeatedly create receipts, share them, and get closure outcomes—forming a habit.

Kill thresholds (Day 60):

Week-4 retention <25% 
OR
Receipt creation rate <40% (among onboarded users)
OR
Share-through rate <60% (receipts shared to both parties)

Definitions / measurement:
	•	Onboarded user: created ≥1 intro in product or completed first flow
	•	Receipt creation rate: % onboarded users who create ≥1 receipt
	•	Share-through rate: % receipts that are shared to ≥2 recipients (both parties)
	•	Week-4 retention: users active in week 4 / users active in week 1
	•	Sample size: ≥150 onboarded users
	•	Timeline: Evaluate at Day 60

Why this kills (not pivots):
If users won’t create + share receipts, the protocol doesn’t spread. No loop, no distribution, no moat.

Escape hatch (only if ALL true):
	1.	Receipt creation ≥55%
	2.	Share-through ≥70%
	3.	Retention low due to seasonality/low intro frequency verified by user logs
→ Extend to Day 90 once. If still below thresholds → KILL.

⸻

KILL #2: “CLOSURE VALUE” DOESN’T MATERIALIZE

Hypothesis being tested:
Truferral delivers measurable value by turning introductions from “black holes” into closed outcomes.

Kill thresholds (Day 45–75 window):

14-day closure rate <25%
OR
Median time-to-first-closure >21 days
OR
Net Promoter-like signal: <30% say “I’d use this on my next intro”

Definitions / measurement:
	•	Closure event: receipt reaches terminal state (Met / Completed / Not happening)
	•	14-day closure rate: % receipts with terminal state within 14 days of creation
	•	Time-to-first-closure: from receipt creation → first terminal state (median)
	•	User value signal: 1-question in-product prompt after 2 receipts:
	•	“Would you use Truferral on your next intro?” (Yes/No)
	•	Sample size: ≥400 receipts OR ≥150 onboarded users
	•	Timeline: start measuring Day 15; decide by Day 75

Why this kills (not pivots):
If outcomes don’t close, the receipt becomes “just another link.” No habit, no retention, no protocol.

Allowed fixes BEFORE the kill decision:
	•	simplify states (fewer buttons)
	•	improve reminders (opt-in, non-annoying)
	•	improve receipt page clarity
If still fails → KILL.

⸻

KILL #3: TRUST / SAFETY BACKLASH

Hypothesis being tested:
Receipts and outcome tracking can exist without coercion, shaming, or workplace harm.

Kill thresholds (immediate if triggered):

>8% of creators delete receipts citing privacy/trust
OR
>3 verified coercion/weaponization incidents
OR
>2% of active users file “pressure/shaming” reports

Definitions / measurement:
	•	Trust delete rate: % creators deleting receipts + exit reason = privacy/trust
	•	Weaponization incident (verified): coercion (“show me proof”), workplace retaliation, public shaming with screenshots tied to product
	•	Reports: in-app “report misuse” tagged and reviewed within 24 hours
	•	Timeline: continuous; immediate kill if thresholds hit

Why this kills (not pivots):
If the product causes harm or coercion dynamics, category becomes ethically toxic and reputation damage is permanent.

No escape hatch.

🔄 THREE PIVOT TRIGGERS (MAX 2 TOTAL)

If these hit → pivot strategy (do not kill), but only two pivots allowed.

⸻

PIVOT #1: WRONG BEACHHEAD SEGMENT

Trigger (Month 2–3):

Top retained cohort is NOT our intended beachhead
OR
Intros/user/month in target segment < expected baseline

Pivot options:
	•	Recruiters → founders → sales → community operators (choose the cohort with best closure + repeat receipts)
	•	Reposition messaging + onboarding to that segment

⸻

PIVOT #2: DISTRIBUTION LOOP WEAK (RECIPIENTS DON’T CONVERT)

Trigger (Day 60–90):

Recipient-to-creator conversion <5%

Pivot options:
	•	Make receipt page deliver standalone value to recipients (clear next steps, scheduling link, reminders)
	•	Add “one-tap create your own receipt” CTA
	•	Add lightweight templates for sending intros with receipt

⸻

PIVOT #3: MONETIZATION FAILS (LATER)

Trigger (Month 6, after workflow features exist):

Paid conversion <4% 
AND
Teams/seat expansion not emerging

Pivot options:
	•	Team tier (recruiting/sales)
	•	Seat-based pricing
	•	“Outcome reporting” for teams
	•	Or stay free while scaling protocol
