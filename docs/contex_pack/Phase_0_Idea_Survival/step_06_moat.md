0) THE RULE (NON-NEGOTIABLE)

A real moat must:
	1.	Survive AI acceleration
	2.	Improve with scale
	3.	Compound over time
	4.	Be hard to retrofit by incumbents
	5.	Make copying painful / slow / incomplete
	6.	Become stronger the more it’s used

✅ If it doesn’t compound, it’s not a moat. It’s a feature.

⸻

✅ PRIMARY MOAT (HYPOTHESIS)

Moat Type: Cross-Channel Outcome Graph + Proof History (Receipts Moat)

Precise Definition

A continuously growing intro outcome dataset created by universal receipt links that work across every messaging channel, producing a unique “Outcome Graph”:
	•	Who introduced whom
	•	When
	•	Through which channel
	•	What outcome happened
	•	Time-to-closure
	•	Follow-through patterns over time

This creates personal + organizational proof that cannot be rebuilt quickly.

	•	WhatsApp

That becomes:

✅ MOAT DEPENDENCIES (CRITICAL)

This moat only exists IF these hold true:

⸻

Dependency #1: Receipt Adoption Rate

People must actually use the receipt habit.

✅ Success Target (Phase 1):
	•	≥40% of interviewed users say they would use it
	•	≥30% run a real intro with it within 7 days (Step 8)

❌ If nobody wants to paste a link → moat dies.

⸻

Dependency #2: Low-Friction Updates

The recipient must be able to update in 1 tap.

✅ Target:
	•	≥35% receipts get status updated within 7 days

If updates require logging in, typing, or effort → fails.

⸻

Dependency #3: Trust / Safety Must Hold

Because this touches reputation.

✅ Must be:
	•	opt-in
	•	non-shaming
	•	private by default
	•	no public “scores”

If users feel it’s “tracking people like a credit score” → backlash risk.
